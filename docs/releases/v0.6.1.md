# Release v0.6.1

**Date:** 2026-01-31
**Feature:** F031 - Migrate Core Exceptions to SDPError

---

## Overview

Core custom exceptions now inherit from SDPError with structured fields (category, remediation, docs_url, context). Error messages are actionable with clear guidance.

---

## New Features

### F031: Migrate Core Exceptions to SDPError

**Что нового:**
- WorkstreamParseError, CircularDependencyError, MissingDependencyError → SDPError
- ModelMappingError, ContractViolationError, HumanEscalationError → SDPError
- format_terminal() and format_json() methods on SDPError base
- All exceptions include ErrorCategory, remediation steps, docs_url

**Использование:**

```bash
# Structured error output
from sdp.core.workstream import WorkstreamParseError
error = WorkstreamParseError("Invalid ws_id", file_path=Path("ws.md"))
print(error.format_terminal())  # Terminal with emoji, remediation
print(error.format_json())     # JSON for CI/CD parsing
```

---

## Breaking Changes

None. All exception class names preserved. Inheritance from Exception maintained.

---

## Migration Guide

No migration required. Existing `except WorkstreamParseError` blocks work unchanged.
